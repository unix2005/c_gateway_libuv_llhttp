## 优化建议

基于当前测试结果，如需进一步提升性能，可考虑：

### 短期优化
1. **启用 Keep-Alive**: 减少 TCP 握手开销
2. **调整 worker 线程数**: 根据 CPU 核心数优化（当前配置可能未达最优）
3. **SO_REUSEPORT**: 已在代码中启用，确保内核正确分发连接

### 中期优化
1. **HTTP/2 支持**: 提升多路复用能力
2. **连接池**: 向后端服务发起请求时使用连接池
3. **零拷贝**: 使用 `sendfile()` 等系统调用减少数据拷贝

### 长期优化
1. **DPDK/SPDK**: 用户态网络栈，突破内核瓶颈
2. **RSS 多队列**: 利用网卡多队列硬件特性
3. **NUMA 感知**: 在多 NUMA 节点服务器上优化内存访问
